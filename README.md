# NetworkManager-pymodule

## Описание

`NetworkManager-pymodule` — Python-модуль для управления NetworkManager через D-Bus.

Модуль предоставляет набор синхронных методов, которые могут вызываться через асинхронный слой (`asyncio.to_thread`). 
По факту в cli реализован  асинхронный вызов не асинхронных методов напрямую. Не задействую асинхронную прокладку, которая предусмотрена в модуле.

Поддерживается работа только с IPv4.

---

## Зависимости

- dbus-python==1.3.2
- pytest

## Основные методы модуля

### get_profile
Получение текущей конфигурации сетевого интерфейса:
- IP адрес
- prefix
- gateway
- DNS
- режим (DHCP / static)

---

### edit_profile
Полная настройка профиля:
- IP адрес
- prefix (или маска)
- gateway (опционально)

Рекомендуется использовать первым, если профиль пустой.

---

### set_ip
Изменяет только IP адрес.
Остальные параметры (prefix, gateway, DNS) не затрагиваются.

---

### set_prefix
Устанавливает prefix сети.

Поддерживает два варианта ввода:
- числовой prefix (например `24`)
- маска подсети (например `255.255.255.0`)

Если передана маска — она автоматически конвертируется в prefix.

---

### add_dns
Добавляет DNS сервер к существующей конфигурации.

---

### enable_dhcp
Переключает интерфейс в DHCP режим.

При успешном применении настройки подтягиваются автоматически с DHCP сервера.

Если DHCP недоступен, профиль может оказаться пустым.

---

## Основные проверки в модуле

Внутри реализованы следующие типы проверок:

### Проверки интерфейса
- интерфейс должен существовать
- интерфейс должен быть управляемым NetworkManager

### Проверки IP адреса
- валидность IPv4 адреса
- корректность формата (нельзя строки или битые значения)
- проверка публичных/частных адресов (в зависимости от сценария)

### Проверки prefix / mask
- prefix должен быть в диапазоне 0–32
- маска должна быть корректной IPv4 маской
- маска должна быть непрерывной (например `255.255.255.0` допустима, `255.0.255.0` — нет)
- некорректные строки должны вызывать ValueError

### Проверки DNS
- предотвращение дублей DNS
- проверка формата IP адреса DNS

### Проверки gateway
- gateway должен быть в той же подсети, что и IP
- предупреждение при конфликте с default route

### Проверки DHCP
- корректное переключение режима DHCP
- очистка статических параметров при включении DHCP

## Makefile цели

### prepos_local

Подготавливает тестовую среду:

- удаляет старые тестовые соединения (`test0`, `test1`)
- создаёт виртуальные интерфейсы `test0` и `test1`
- один интерфейс отдаётся под управление NetworkManager
- второй остаётся unmanaged

После выполнения:
- интерфейсы существуют
- профиль может быть создан, но без конфигурации

---

### reinstall

Полный цикл переустановки модуля:

- удаляет установленную версию пакета
- очищает артефакты предыдущих сборок
- собирает wheel через Docker
- устанавливает пакет из `dist_out`
- проверяет импорт модуля

---

### test-smoke

Быстрая проверка основных сценариев работы через CLI:

Проверяются реальные операции:

- получение профиля
- установка IP
- изменение prefix
- добавление DNS
- включение DHCP

Используется для проверки, что модуль “живой” и работает end-to-end.

### test-units

### Тесты покрывают следующие ситуации:

#### Работа с интерфейсами
- интерфейс не управляется NetworkManager (RuntimeError)
- интерфейс не существует (ValueError)
- предупреждение при изменениях на default route (логическое поведение, без падения)
- внесение первичных данных профиля через edit-profile (IP + GW + prefix)

---

#### IP адрес
- корректная установка IPv4 адреса
- отклонение некорректного формата IP (битые строки)
- отклонение публичных IP (разрешены только private сети)
- отклонение IP версии IPv6 (не поддерживается)
- отклонение адреса сети (например 192.168.1.0)
- отклонение broadcast адреса (например 192.168.1.255)
- сохранение остальных параметров при изменении IP

---

#### DNS
- добавление DNS сервера
- защита от дубликатов DNS записей
- отсутствие повторного применения конфигурации при дубле

---

#### DHCP
- включение DHCP режима
- корректное переключение статического → DHCP профиля
- очистка или перезапись IPv4 конфигурации при DHCP

---

#### Prefix / Mask
- корректная работа с числовым prefix (например /16)
- корректная работа с маской подсети (например 255.255.255.0)
- обработка prefix = 0 (запрещён)
- обработка слишком маленького prefix для приватной сети
- отклонение некорректных масок (битые или не IPv4 формат)
- отклонение не непрерывных масок (например 255.0.255.0)
- предупреждение при /32 (один хост)

---

### Что пока НЕ покрыто тестами (но логически присутствует в коде)

#### Gateway validation
- проверка, что gateway находится в той же подсети (в `_validate_ip`)
- сейчас поведение проверяется косвенно (warning/print), но нет assert-теста

---

#### Расширенные edge cases
- IPv6 адреса явно не тестируются (только через ValueError косвенно)
- смешанные конфигурации (IPv4 + некорректный gateway)
- наличие предупреждения что работаем с default route при смене сетевого профиля

---

#### Mask / prefix пограничные сценарии
- частично не тестируется поведение при граничных масках типа:
  - /31
  - /32 (есть warning, но не полностью проверено поведение сети)
